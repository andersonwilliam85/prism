# Startup Config

**For:** Fast-moving startups (10-50 employees)  
**Example:** YC startups, seed/Series A companies  
**Hierarchy:** Flat (just teams)  

## What's Included

- **Flat structure:** No bureaucracy!
- **Minimal fields:** Name, email, GitHub, Slack
- **Move fast:** Ship on day 1
- **Modern stack:** Latest tools and frameworks
- **No approvals:** Direct access to everything

## User Info Fields

- Name (just first name!)
- Email
- GitHub Username
- Slack Handle
- Role (optional)

## Structure

```
Startup (base)
└── Teams
    ├── Engineering
    └── Product
```

**That's it! No complex hierarchy.**

## Installation

```bash
python3 scripts/package_manager.py install startup-config
python3 install-ui.py
```

## Philosophy

- **Move fast:** No waiting for approvals
- **Everyone codes:** Engineers ship to production
- **Full access:** Trust by default
- **Modern tools:** Latest and greatest
- **Simple:** If it's complicated, we're doing it wrong
